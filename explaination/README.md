# مشروع Digital Cave (البيئة الكريستالية)

## نظرة عامة

مشروع **Digital Cave** هو تجربة تفاعلية ثلاثية الأبعاد (3D) مبنية باستخدام مكتبة **Three.js**، تهدف إلى تقديم المعرفة بطريقة بصرية immersive. يشبه المستخدم أنه يطير داخل كهف بلوري (كريستالي) عملاق، وكلما تعمق أكثر يواجه فروعًا علمية مختلفة يمكنه النقر عليها لاستكشاف المزيد.

---

## هيكل المشروع

```
digital-cave/
├── index.html          # صفحة المدخل الرئيسي (الواجهة)
├── sections.html       # صفحة الأقسام الفرعية لكل مجال
├── posts.html          # صفحة المقالات داخل كل قسم
├── styles/
│   └── main.css        # الأنماط والتنسيقات العامة
├── js/
│   ├── config.js       # إعدادات الفروع وحدود الحركة
│   ├── geometry.js     # دوال إنشاء الهندسة الكريستالية
│   ├── scene.js        # إنشاء المشهد والكاميرا والإضاءة
│   ├── animate.js      # حلقة التحريك المستمرة
│   ├── controls.js     # التحكم في الحركة والاختيار
│   ├── ui.js           # عناصر واجهة المستخدم (اللوحات)
│   └── main.js         # نقطة البداية (الربط بين كل المكونات)
└── explaination/
    └── ...             # هذا المجلد - شروحات لكل ملف
```

---

## تدفق العمل (User Flow)

1. **index.html** → يفتح كهفًا طويلًا بثلاثة فروع جانبية: العلوم الطبيعية، الاجتماعية، الشكلية
2. **sections.html** → عند النقر على فرع، يدخل المستخدم إلى ممر يعرض أقسامًا فرعية لذلك المجال
3. **posts.html** → عند النقر على قسم فرعي، يرى المستخدم لوحات لمقالات داخل ذلك القسم

---

## التقنيات المستخدمة

- **Three.js r128** — مكتبة الرسم ثلاثي الأبعاد
- **JavaScript ES Modules** — تنظيم الكود في وحدات مستقلة
- **Canvas 2D API** — رسم النصوص والأنماط على اللوحات
- **CSS3** — التنسيقات البصرية والانتقالات

---

## كيفية التشغيل

المشروع يعمل بالكامل في المتصفح.只需 فتح `index.html` في أي متصفح حديث. لا يحتاج إلى خادم (server) لأن الوحدات تُحمّل محليًا، لكن بعض المتصفحات قد تطلب تشغيله عبر خادم محلي بسيط:

```bash
# باستخدام Python
python3 -m http.server 8000
# ثم افتح http://localhost:8000
```
